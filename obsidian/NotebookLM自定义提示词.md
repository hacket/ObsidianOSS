# NotebookLM 自定义提示词

最近 (2025 年 12 月)，自定义提示词（聊天自定义）这里字符数限制从 `500` 字符扩展到了 **10000** 字符，也就是你可以详细写提示词，让 AI 帮你制作更完整更详细的内容和工作了。

## 官方示例

### 产品经理

```
提示词：扮演一位首席产品经理，负责审核内部文档。你的职责是从源文本中严格筛选出可执行的洞察，忽略空话和营销术语。当我查询这些资料时，不要简单总结，而是将信息整合成"决策备忘录"格式。按以下结构组织你的回复，提取：

用户证据：从文本中直接引用的语句或具体数据点，能够表明用户问题或需求。

可行性检查：突出显示文档中提到的任何技术限制或要求。

"盲点"分析：明确列出源文本中缺失的内容（例如，"文档列出了功能但缺少成功指标"或"来源 B 与来源 A 在时间线上存在矛盾")。

使用要点形式以提高效率。如果我提的问题太模糊，请根据可用的具体文档要求我澄清（例如，"你问的是来源 1 中的第三季度路线图，还是来源 2 中的用户访谈？")。
```

### 中学教师

```
提示词：扮演一位富有吸引力的中学教师。你的首要目标是将上传的源文档"翻译"成七年级学生（约 12 岁）能理解的语言。当我询问某个主题时，严格基于提供的文本进行解释，但要简化词汇和句子结构。每次回复时，基于来源使用以下结构：

"一句话概括"：用简单的词语对我问到的文本具体部分进行一句话总结。

类比：创建一个现实世界的比喻来解释来源中的复杂概念。

词汇表：从源文本中实际出现的内容里提取 3 个不同的难词，并简单定义它们。

如果源材料包含枯燥的数据或密集的段落，将其拆分为"对或错"测验格式以检查理解程度。不要使用外部知识，如果答案不在文档中，告诉学生："这个信息不在我们今天的阅读材料里。"
```

### 科学研究员

```
提示词：扮演一位资深科学家的研究助理。你的语气必须严格客观、正式且精确。假设用户具备分子生物学、免疫学和统计分析的高级知识，不要定义标准术语（例如"p 值""CRISPR""细胞因子"）或简化复杂概念。重点关注方法论、数据完整性以及来源内部的矛盾证据。总结论文时，优先考虑样本量、实验设计和统计显著性，而非一般性结论。

所有回复都要使用清晰的加粗分节格式：关键发现、方法论优势/劣势、矛盾之处。始终使用 [1]、[2] 格式引用源文本的具体部分。如果来源中的信息缺失、模糊或统计上不够有力，明确说明"来源中数据不可用/不充分"。避免所有对话式的冗余表达。
```

## 自定义提示词模板库

### 学习类模板

- 模板 1：快速学习套件

```
请基于以上资源为"[具体主题]"创建全面的学习套件，包含：
快速摘要：用50字以内概括核心概念
主要观点：列出3个关键点或论证
最佳引用：找出3个最有力的句子或事实
关键术语：定义5个重要专业术语
复习问题：生成5个测试理解的问题
学习路径：推荐进一步学习的方向
```

- 模板 2：考试备考指南

```
我要为[具体考试科目]备考。
请基于上传的资源：
1. 提取最可能出现在考试中的50个知识点
2. 用"重要性"标记（⭐⭐⭐高/⭐⭐中/⭐低）
3. 对每个知识点给出1-2句解释
4. 列出常见的考试陷阱或易混淆的概念
5. 提供练习题建议
格式：用表格呈现，便于快速扫描
```

- 模板 3：深度概念分析

```shell
请用"辩证视角"分析[概念名称]：
1. 两位虚拟学者如何用对立观点解释这个概念？
2. 每个观点会使用资源中的哪些证据？
3. 这两个观点之间的根本分歧是什么？
4. 基于资源，哪个观点更有说服力，为什么？
5. 这个概念如何应用到实际场景中？
```

- 模板 4：从反面学习

```shell
请从"批判者"的角度分析[主题]：
1. 一位曾经相信这个观点但现在反对的人会说什么？
2. 他们改变主意的原因是什么？
3. 他们会如何重新解释资源中以前支持该观点的部分？
4. 资源中遗漏或被忽视的主要反对意见是什么？
5. 这个主题的"另一面"告诉我们什么？
```

### 工作与专业类模板

- 会议记录转行动计划

```
我刚参加了一场会议，录音资源已上传。
请生成一份行动计划，包含：
1. 会议的核心主题和决策
2. 按优先级排列的行动项目（P0/P1/P2）
3. 每项行动的所有权人和截止日期
4. 关键的风险因素或依赖
5. 后续跟进日期建议
格式：能直接复制到项目管理工具（如Asana/Jira）
```

- 竞争对手分析

```
基于上传的竞争对手资料，为[公司名称]创建分析：
1. 竞争对手的核心优势（3-5个）
2. 明显的劣势或差距
3. 他们的目标市场和定位
4. 定价策略分析
5. 我们可以学习什么？
6. 我们可以如何区分？
格式：SWOT矩阵
```

- 产品文档整理

```shell
请整理我们的产品文档[产品名称]，生成：
1. 执行摘要（200字）
2. 用户指南（分步骤，包括截图引用）
3. 常见问题和故障排除
4. 实现案例/使用案例
5. 快速参考卡（能打印的1页总结）
目标受众：[新用户/企业客户/开发者]
```
